# PagePost: 웹페이지 컨텍스트 메모 시스템 제품 기획서 (PRD)

## 1. 개요 (Executive Summary)
PagePost는 웹 탐색 중 발견한 정보를 "그 자리에" 포스트잇 형태로 남길 수 있는 크롬 확장프로그램입니다. 기존 메모 도구들이 맥락을 단절시키는 문제를 해결하며, Spatial Memory(공간 기억)를 활용해 웹 리서치와 학습의 효율을 극대화합니다.

## 2. 제품 정의 및 비전
- **비전**: "웹페이지를 읽는 공간에서, 바로 생각을 붙인다."
- **핵심 가치**:
  - **Zero Context Switch**: 페이지 이탈 없이 즉각적인 메모.
  - **Spatial Memory**: 위치 기반 고정으로 시각적 맥락 유지.
  - **Intelligent Restoration**: 페이지 변경에도 유지되는 3중 앵커링 시스템.

## 3. 핵심 기능 명세

### 3.1 포스트잇 생성 및 관리
- **생성**: 우클릭 메뉴 또는 단축키 (`Ctrl+Shift+N`)를 통해 커서 위치에 즉시 생성.
- **인터랙션**: 드래그로 위치 이동, 우하단 핸들로 리사이즈 (150px~600px).
- **표시 모드**:
  - **접힌 상태**: 30x30px 아이콘 (비침습적 디자인).
  - **펼친 상태**: 전체 내용 메모 카드.
  - **편집 모드**: 더블클릭 시 텍스트 편집 (마크다운 지원).

### 3.2 3중 앵커링 시스템 (Triple Anchor System)
페이지 레이아웃 변화에 대응하기 위한 위치 복구 알고리즘:
1. **1차 (DOM Selector)**: CSS 선택자 및 XPath 기반 정확한 매칭.
2. **2차 (Text Context)**: 앞뒤 50자 텍스트 패턴 매칭.
3. **3차 (Coordinate)**: 뷰포트 상대 좌표 및 스크롤 위치 기반 배치.

### 3.3 분류 및 동기화
- **색상 팔레트**: Butter Yellow (일반), Mint (아이디어), Soft Pink (중요), Lavender (TODO), Sky Blue (쇼핑).
- **검색/태그**: 전체 메모 검색 및 `#태그` 자동 인식.
- **동기화**: 로그인 시 Supabase를 통한 다중 기기 실시간 동기화.

## 4. 사용자 시나리오
- **리서치/쇼핑**: 여러 쇼핑몰의 가격과 특징을 해당 상품 페이지 요소 옆에 바로 메모하여 비교.
- **학습/연구**: 기술 문서의 특정 섹션 옆에 보충 설명이나 나중에 읽을 포인트를 메모 및 핀핑.

## 5. UI/UX 디자인 원칙
- **비침습성**: 페이지 가독성을 해치지 않도록 평소엔 최소화.
- **즉각성**: 단 한 번의 액션으로 메모 생성 가능.
- **가시성**: 필요할 때만 선명하게 표시되는 인터랙션.
