## 1. 제품 개요 (Product Overview)

### 1.1 제품명
**PagePost Markup (가격: PagePost)**

### 1.2 한 줄 정의
"웹페이지 위 원하는 위치에 포스트잇을 붙이고, 펜·형광펜·도형·화살표로 즉시 표시할 수 있는 **브라우저 기반 주석(Annotation) 워크스페이스**"

### 1.3 핵심 가치 (Core Value)
기존의 메모 앱이 "내용"만을, 그리기 툴이 "표시"만을 파편화되어 제공했다면, **PagePost Markup**은 이 둘을 결합하여 **"왜 중요한지(Context)"**와 **"어디가 중요한지(Position)"**를 하나의 레이어에서 동시에 관리하도록 돕습니다.

### 1.4 제품 비전 (Product Vision)
사용자가 웹페이지를 단순히 소비하는 수동적 관찰자에서 벗어나, 웹 위에 자신만의 생각과 표시가 남는 **"개인 작업 레이어(Personal Layer)"**를 구축하고 영구적으로 소유하도록 만드는 것을 목표로 합니다.

---

## 2. 해결하려는 문제 (Problem Statement)

웹 탐색 중 중요한 발견 시 발생하는 기존의 번거로운 프로세스를 혁신합니다.

| 기존 방식 (As-Is) | 발생하는 문제점 (Pain Points) | PagePost Markup (To-Be) |
| :--- | :--- | :--- |
| 메모 앱에 따로 적기 | 문맥 단절 (Context Switching) | 웹페이지 **그 자리**에 즉시 기록 |
| 캡처 후 이미지에 표시 | 위치 정보 유실 (Static State) | 실제 페이지 위 **라이브 아카이빙** |
| 협업 도구로 링크 전달 | 설명 비용 발생 (Wait for Sync) | 마크업이 포함된 **컨텍스트 공유** |
| 머릿속으로만 기억 | 망각 및 검색 불가 | **스마트 앵커링**으로 재방문 시 복원 |

---

## 3. 제품 포지셔닝 (Product Positioning)

단순한 포스트잇 앱을 넘어, **웹페이지 환경에 최적화된 마크업 + 검수 코멘트 도구**로 정의합니다.

> [!IMPORTANT]
> **포지셔닝 선언**: "읽는 곳에서 바로 기록하고, 설명 대신 표시로 남기는 웹페이지 위 나만의 작업 레이어."

### 주요 경쟁 영역 및 차별점
- **메모 툴**: 텍스트 데이터 그 이상의 **공간적 맥락(Spatial Context)** 제공.
- **드로잉 툴**: 단순한 선을 넘어 **메모와의 데이터 연결성** 확보.
- **QA 도구**: 전문 디자인 툴보다 가볍고 빠른 **실시간 라이브 피드백** 최적화.

## 4. 타깃 사용자 분석 (Target Audience)

1. **개인 생산성 사용자 (Single Player)**
   - 기획자/제작자: 레퍼런스 수집 및 분석.
   - 학생/리서처: 논문 및 기술 문서 학습용 주석.
   - 쇼핑러: 복수 상품의 상세 옵션 비교.
2. **리뷰 및 피드백 사용자 (Reviewer)**
   - PM/QA/디자이너: 사이트 개선점 포착 및 수정 요청.
   - 마케터: 랜딩 페이지 AB 테스트 포인트 기록.
3. **협업 팀 (Collaborators)**
   - 스타트업 제품팀: 기획안 리뷰 및 개발 협의.
   - 에이전시: 고객사와의 컨펌 및 피드백 루프.

## 5. 핵심 기능 구조 (System Architecture)

### A. 스마트 포스트잇 시스템
- **컨텍스트 생성**: 우클릭/단축키 기반 타겟 요소 앞 생성.
- **동적 제어**: 드래그 이동, 자유 리사이즈, 색상 팔레트 적용.
- **상태 관리**: 메모 접기/펼치기, 핀 고정, 완료(Done) 처리.
- **데이터 바인딩**: 현재 페이지 요소와 메모 데이터의 강한 결합.

### B. 프리시전 마크업(Markup) 시스템
- **드로잉 도구**: 펜, 형광펜, 사각형, 원형, 화살표, 텍스트 라벨.
- **편집성**: 지우개, Undo/Redo, 개별 요소 선택 삭제.
- **가시성**: 선 두께 및 투명도 조절, 강조 모드.

### C. 통합 리뷰 및 관리 시스템
- **멀티 레이어 뷰**: 메모와 드로잉 통합 렌더링.
- **지능형 필터링**: 현재 페이지 / 도메인 / 태그 / 날짜 별 검색 및 필터.
- **리뷰 대시보드**: 전체 작업물 관리 및 협업용 링크 생성.

## 6. 제품 핵심 차별화 전략 (Differentiators)

1. **메모-드로잉 마그네틱 결합 (Linked Context)**
   - 포스트잇과 특정 마크업(화살표 등)을 그룹화하여, 이동 시 함께 움직이고 클릭 시 동시 강조됨.
2. **3중 앵커링 기반 라이브 복원 (Triple Anchoring)**
   - 스냅샷이 아닌 실제 DOM 위에서 동작하며, 사이트 개편 시에도 메모 위치를 복원하는 지능형 알고리즘.
3. **비침습적 UX (Non-Invasive design)**
   - 평소에는 거의 보이지 않는 'Clean View' 상태를 유지하다가, 필요 시에만 UI가 확장되는 구조.
4. **라이브 레이어 (Live Layer Technology)**
   - 원본 웹페이지의 동작을 방해하지 않으면서 그 위에 투명한 작업 창을 띄워 실시간 Interaction 유지.

---

## 7. 위치 저장 기술 (Anchoring Strategy)

웹페이지 변화에 대응하기 위한 당사만의 **3중 하이브리드 복원 방식**을 제안합니다.

1. **Precision Anchor (DOM Selector)**: ID, CSS Selector, XPath를 통한 정적 위치 매칭.
2. **Contextual Anchor (Text Matching)**: 주변 텍스트 50~100자의 패턴을 분석하여 위치 유추.
3. **Relative Anchor (Viewport/Coordinate)**: 부모 요소 대비 상대 % 좌표 및 뷰포트 비율 기반 보정.

---

## 8. 모드 설계 (Mode Design)

| 모드명 | 주요 목적 | 인터랙션 특징 |
| :--- | :--- | :--- |
| **Note Mode** | 기록 및 생각 정리 | 포스트잇 생성과 텍스트 입력에 집중 |
| **Markup Mode** | 시각적 강조 및 지시 | 드로잉 툴 활성화, 공간적 표시 기능 |
| **Review Mode** | 작업물 검토 및 공유 | 요소별 필터링, 대시보드 리스트 확인 |
| **Clean View** | 원본 페이지 서핑 | 레이어는 유지하되 투명도/아이콘화로 숨김 |

## 9. 비즈니스 로직 및 로드맵

### 9.1 수익 모델 (Monetization)
- **Free**: 로컬 저장 중심, 기본 마크업, 기능 개수 제한.
- **Pro**: 무제한 클라우드 동기화, 고급 필터, 스냅샷 백업, 대시보드.
- **Team**: 워크스페이스 공유, 댓글, 담당자 지정, 리뷰 승인 프로세스.

### 9.2 로드맵
- **Phase 1 (MVP)**: 메모 + 기본 마크업 + 로컬 저장 및 유효한 위치 복원.
- **Phase 2 (Efficiency)**: 동기화, 도메인별 뷰어, 템플릿 메모, 단축키 확장.
- **Phase 3 (Collaboration)**: 공유 링크, 스레드형 댓글, 리뷰 상태 관리 (B2B 진입).
- **Phase 4 (AI Layer)**: AI 자동 요약, 관련 메모 추천, 중요 영역 자동 하이라이트.

## 10. 리스크 관리 (Risk Management)

- **사이트 구조 변경**: 3중 앵커링 고도화 및 복원 실패 시 근접 위치 제안 UI 제공.
- **성능 부하**: Canvas Virtualization 기술을 통한 대량 마크업 시 렉 방지.
- **UX 복잡도**: 기본 모드를 'Note'로 고정하고 'Markup'은 모달/툴바형태로 필요한 시점에만 노출.
